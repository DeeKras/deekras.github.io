Some simple recursive functions

Wed 17 December 2014 by Dee Kras

So how do recursions work?

I hear so much about recursions. They're supposedly in all interview questions. So here's my attempt at trying to make them easy to understand.

A recursive function is a function that calls itself. I'll throw in a simple example. The Fibonacci.

the fibonacci of 5 = 5 x 4 x 3 x 2 x 1 what is happening here is: - first, 5 is multiplied by 5-1 (in other words 4) = 20 - then, that result is multiplied by 4-1 (3) = 60 - then, that result is multiplied by 3-1 (2) = 120 - lastly, that result is multiplied by 2-1 (1) = 120 - and it's over. we have the result.

There's a repetetive pattern here - the number is multiplied by (the number - 1) n * (n-1). That's where the recursive function comes in.

def fibonacci(n):
    if n <= 1:
        return 1
    else:
        return n * fibonacci(n-1)

So what is happening here?

Let's look at the last line. return n * fibonnaci(n-1). the number is being multiplied by ... the result of the same function. In other words a loop of the function calling itself.

Let's look at the construct of the rest of the function and come back to the last line. We start off with if n <= 1:. Every recursive function must have an end to the loop, or it will go on until Python stops it. So in this case, if the number is less than or equal to 1, stop the function by return 1.
